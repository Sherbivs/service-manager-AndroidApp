# В§ 1 вҖ” System Overview

**Service Manager** is a lightweight, zero-dependency-beyond-Express service orchestrator with a web dashboard for managing local development services.

## Technology Stack

| Layer | Technology | Notes |
|-------|-----------|-------|
| Runtime | Node.js | No build step, vanilla JS |
| Server | Express 4.x | Static file serving + JSON API |
| Frontend | HTML/CSS/JS | No framework, no bundler |
| Process Mgmt | child_process (spawn) | Native Node.js process control |
| Config | JSON file | `services.json` вҖ” hot-reloaded per request |

## Key Invariants

1. **Config is hot-reloaded** вҖ” `services.json` is read on every API request, no restart needed to add services.
2. **Stateless server** вҖ” The only runtime state is the in-memory process registry (`Map`). No database.
3. **Network accessible** вҖ” Binds to `0.0.0.0`, reachable from any device on the LAN.
4. **Auto-restart** вҖ” Services with `autoRestart: true` are restarted on crash with configurable delay.
5. **Health-aware** вҖ” Services with `healthCheck` URLs are polled to determine actual running status.
